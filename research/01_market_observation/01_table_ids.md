## 0. 이 문서의 위치

채영님이 풀세트(7개 지표)를 선택했고 인증키도 발급받으셨다. 이 문서는 7개 지표를
KOSIS 통계표 ID에 매핑한다. **확정**과 **첫 실행 시 검증 필요**를 분명히 구분한다.

KOSIS는 통계표를 조용히 개편하는 경우가 있다. 미리 ID를 100% 매핑하는 것보다,
**코드 안에 통합검색 헬퍼를 두고 첫 실행 때 검증하는 방식**이 견고하다. `02_collector.py`
가 그렇게 짜여 있다.

---

## 1. 핵심 식별자

- **기관 ID (orgId)**: `101` (통계청 / 국가데이터처)  
  ※ 통계청은 2025년 정부조직 개편으로 "국가데이터처"가 됐지만 KOSIS의 orgId는
  그대로 `101`을 유지한다.
- **시점 단위 (prdSe)**: `Y`(연), `Q`(분기), `M`(월)
- **수집 시작 연도 기본**: 2010년 (시계열 15년 확보)

## 2. 7개 지표 매핑

### 지표 1. 연도별·시도별 사망자 수 (인구동향)

| 항목 | 값 | 상태 |
|---|---|---|
| 통계표 ID | `DT_1B8000G` | ✅ 확정 |
| 통계표명 | 월·분기·연간 인구동향(출생,사망,혼인,이혼) | |
| orgId | 101 | |
| prdSe | M / Q / Y 모두 가능. 우리는 `Y` | |
| 분류 (objL1) | 시도 (ALL로 전체 수집) | |
| 항목 (itmId) | 사망자 수 항목코드 — **첫 실행 시 검증** | ⚠️ |

**비고**: 이 표는 출생·사망·혼인·이혼 종합표라 itmId 필터를 줘야 사망자수만 추출
가능. `02_collector.py`의 `verify_items()`로 첫 실행에서 itmId를 자동 출력.

## 4. 첫 실행 절차

1. `pip install PublicDataReader python-dotenv pandas`
2. `.env`에 `KOSIS_API_KEY=발급받은키` 추가
3. `python 02_collector.py --verify` 실행
   → 각 지표에 매핑한 통계표 ID가 실제로 존재하는지, 응답 구조가 어떤지 출력
   → 어긋난 ID는 통합검색으로 후보 표시
4. 어긋난 ID는 채영님이 화면 출력을 보고 ID 수정 후 다시 실행
5. 통과되면 `python 02_collector.py --collect` 로 실제 수집

이 절차가 끝나면 `data/` 폴더에 7개 CSV가 쌓이고, `snapshots/2026-05-03/`에 동일
파일이 분기 스냅샷으로 보존된다.

## 5. 파라미터 단위 결정 사항

이번 인프라에서 통일한 기본값:

- **시점 범위**: 2010~최신
  - 시계열 15년 이상 확보 → 베이비부머 사망 진입 추세 가시화 가능
  - 그 이전 데이터가 필요해지면 별도 재수집
- **공간 단위**: 시도 (필요시 시군구로 확장)
  - 제주 단독 추출은 시도 데이터에서 필터링
- **CSV 인코딩**: `utf-8-sig`
  - Excel에서 직접 열 때 한글 깨짐 방지
- **결측 처리**: KOSIS 응답 그대로 보존. 가공은 분석 단계에서.
