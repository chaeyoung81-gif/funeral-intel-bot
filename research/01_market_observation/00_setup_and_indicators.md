## 0. 이 문서의 목적

장례 시장의 구조를 정량적으로 관측하기 위한 KOSIS 기반 인프라의 **첫 삽**.
이번 문서에서 합의할 것은 두 가지뿐이다.

1. 채영님이 KOSIS 인증키를 발급받는 절차
2. 어떤 지표 7개를 시계열로 누적할지 — 후보 7개 중 선택과 우선순위

수집 스크립트와 통계표 ID 정확 매핑은 인증키 발급 후 2차 문서로 분리한다.

## 1. 현재 KOSIS API 환경 (2026년 5월 기준 확인)

채영님이 알아두실 사실만:

- **회원가입 + 활용신청**으로 인증키 발급. 회원당 1개, 모든 서비스 공통.
- **API 호출 제한**: 분당 호출건수 제한이 있다 (2026년 2월부터 도입). 일간/월간 총량 제한은 없음.
- **호출 1회당 최대 40,000건** 자료 조회 가능.
- **HTTPS만 사용 가능** (HTTP 프로토콜 제공은 종료됨).
- **자동승인** — 신청 즉시 키 발급. 별도 심사 없음.

비용은 무료. 카드 등록 같은 것 일절 없음.

---

## 2. 채영님이 먼저 해야 할 일 — 인증키 발급

순서대로 따라하시면 됩니다. 5분 내외.

1. **회원가입**
   - https://kosis.kr/openapi/ 접속
   - 우측 상단 회원가입 (이미 KOSIS 회원이면 로그인만)

2. **로그인 후 활용신청**
   - 상단 메뉴 "활용신청" → "Open API 서비스 신청"
   - 활용 목적: "장례 산업 시장 관측을 위한 시계열 데이터 수집" 정도로 쓰면 충분
   - 자동승인 → 인증키 발급

3. **인증키 보관**
   - 발급된 키는 `.env` 파일에 추가:
     ```
     KOSIS_API_KEY=발급받은_인증키
     ```
   - funeral-intel-bot의 `.env`에 함께 두면 됨. 텔레그램 봇과 키 공유 안 해도 충돌 없음.

4. **확인 방법**
   - "개발가이드" → "통계자료" → "URL 생성" 탭에서 임시 통계표 ID 하나 입력
   - 결과값 보기 눌렀을 때 JSON이 정상 반환되면 OK

이 4단계가 끝나면 저에게 "키 발급 완료"라고만 알려주세요. 그 다음 2차 문서로 넘어갑니다.

## 3. 핵심 지표 7개 후보

각 지표마다 **왜 필요한지 / 어디서 가져오는지 / 갱신 주기**를 정리한다.
채영님이 7개 전부를 받을 수도 있고, 우선 4~5개만 좁힐 수도 있다.

### 후보 1. 연도별 사망자 수 (전국·시도별)

- **왜**: 시장 규모의 가장 기본 변수. 한국은 2025년경 연 36~38만 명 → 2040년대
  연 50~60만 명대로 진입 예정. 채영님 사업의 "골든 윈도우"가 정확히 언제 열리는지
  볼 수 있는 단일 지표.
- **출처**: 통계청 인구동향조사
- **공간 분해**: 시도별, 시군구별까지 가능 (제주 단독 추출 가능 → 거점 검증)
- **갱신 주기**: 분기별 잠정치, 연간 확정치
- **우선순위 추천**: ★★★★★ (필수)

### 후보 2. 연령대별 사망자 수·사망률

- **왜**: 1차 베이비부머(55~63년생)와 2차 베이비부머(68~74년생)가 언제 어느 비중으로
  사망 통계에 진입하는지 본다. "어느 세대부터 새로운 장례 수요가 임계점을 넘는가"
  가설 검증의 기초 데이터.
- **출처**: 통계청 사망원인통계
- **갱신 주기**: 연 1회 (전년도 9월경 확정 발표)
- **우선순위 추천**: ★★★★★ (필수)

### 후보 3. 사망 장소별 분포 (의료기관 / 자택 / 사회복지시설 / 기타)

- **왜**: 채영님이 발전시키신 자택 사망 원스톱 플랫폼 가설의 직접 검증 데이터.
  현재 한국은 의료기관 사망 비율 75% 전후. 자택 사망 비중이 어떻게 움직이는지,
  요양시설 사망이 늘어나는지가 진입로 설계에 영향.
- **출처**: 통계청 사망원인통계 (장소별 분포 표)
- **갱신 주기**: 연 1회
- **우선순위 추천**: ★★★★☆ (자택사망 트랙을 살리려면 필수, 영업·중개 트랙만 갈
  거면 ★★★)

### 후보 4. 1인가구 추이 (전국·시도·연령대별)

- **왜**: "장례 결정 주체 공백" 가설의 핵심 데이터. 비혼 1인 가구가 고령화되면
  무연고 사망 + 사전준비 시장 양쪽으로 분기. 일본 종활 시장이 폭발한 시점도
  1인가구 비율 30%를 넘긴 시점과 일치.
- **출처**: 통계청 인구주택총조사 / 인구총조사 등록센서스 (가구 부문)
- **갱신 주기**: 연간 (등록센서스 기반)
- **우선순위 추천**: ★★★★★ (필수)

### 후보 5. 고령자 가구 구성 — 독거노인 비율·무배우자 비율

- **왜**: 후보 4를 고령층으로 좁힌 것. 8050 현상의 한국판 진행 속도 추적.
  부모 사망 시 결정 주체가 누구인지 — 자녀 동거 vs 독거 vs 부부 — 의 분포가
  바로 장례 의사결정 구조의 분포다.
- **출처**: 통계청 인구주택총조사 / 고령자 통계 (매년 9월경 발표되는 종합 보고서)
- **갱신 주기**: 연 1회
- **우선순위 추천**: ★★★★☆

### 후보 6. 장래인구추계 — 사망자 수 피크 시점 추정

- **왜**: 미래 시장 크기의 천장이 어디인지, 그 시점이 언제인지를 봄. 채영님의
  10년·20년 호흡 사업 설계에 필요한 단 하나의 미래 추정 지표. 통계청이 이미
  공식 추계로 제공하므로 자체 모델링 불필요.
- **출처**: 통계청 장래인구추계 (5년마다 갱신, 가장 최근은 2023년 추계)
- **갱신 주기**: 5년 (다음 갱신은 2028년경)
- **우선순위 추천**: ★★★☆☆ (한 번 가져와서 잠재 시장 규모 정해두면 끝.
  자주 갱신할 필요 없음)

### 후보 7. 생명표 — 출생연도별·연령별 기대여명

- **왜**: 1981년생 남성 기대여명, 2차 베이비부머 기대여명 등 코호트별로 "지금부터
  몇 년 더 사는가"를 추적. 사업 타임라인 설계용.
- **출처**: 통계청 생명표
- **갱신 주기**: 연 1회 (12월경)
- **우선순위 추천**: ★★★☆☆ (참고용. 사업 의사결정에 직접 영향은 작음)

## 4. 우선순위 결정 — 채영님이 정하실 것

이번 단계에서 받을 결정은 **이거 하나**입니다.

### 옵션 A. 풀 패키지 — 7개 전부

- 첫 수집 작업량은 늘지만, 한 번 세팅하고 분기별 자동 갱신 돌리면 됨.
- 시장 관측 대시보드가 완성도 높게 시작됨.
- 단점: 처음 셋업에 시간이 더 걸림 (제 작업 기준 +1~2일).

### 옵션 B. 핵심 4개로 좁힘 — 후보 1, 2, 3, 4

- 사망자 수 / 연령별 사망률 / 사망 장소 / 1인가구.
- 이 4개로 진입로 후보 평가의 80%는 가능.
- 후보 5(독거노인), 6(장래추계), 7(생명표)는 필요 시점에 추가.
- **이게 기본 추천**입니다.

### 옵션 C. 미니멈 3개 — 후보 1, 2, 4

- 사망자 수 / 연령별 사망률 / 1인가구.
- 사망 장소 빠지면 자택사망 트랙 평가는 어려워지는데, 채영님이 원장님 루트
  (영업·중개) 우선으로 가실 거면 후보 3을 빼도 무방.
- 가장 가벼운 시작.

추천: **옵션 B**. 이유는, 후보 3(사망 장소)이 빠지면 채영님이 이전에 발전시키신
자택사망 플랫폼 가설을 정량적으로 다룰 근거가 끊긴다. 그 가설을 지금 죽일 필요는
없다고 봅니다. 후보 5·6·7은 나중에 필요할 때 추가하면 됨.

## 부록 A. 왜 PublicDataReader 라이브러리를 쓰는가

KOSIS API를 직접 호출하면 URL 파라미터를 길게 조립해야 하고, 응답 JSON의 한국어
필드명을 매번 파싱해야 한다. PublicDataReader는 한국 개발자(정우일)가 만든 오픈
라이브러리로, KOSIS 공유서비스 API를 pandas DataFrame으로 바로 변환해준다. 안정성
검증된 라이브러리이고, 채영님 봇 인프라(파이썬 기반)에 그대로 얹힌다.

```python
# 설치
pip install PublicDataReader

# 사용 예시 (개념)
from PublicDataReader import Kosis
api = Kosis(service_key="발급받은_인증키")
df = api.get_data(
    "통계자료",
    orgId="101",  # 통계청
    tblId="DT_1B81A21",  # 사망원인통계 (예시 - 2차에서 확정)
    objL1="ALL",
    itmId="ALL",
    prdSe="Y",  # 연 단위
    startPrdDe="2010",
    endPrdDe="2024"
)
```

## 부록 B. 이 문서에서 의도적으로 다루지 않은 것

- **상조업 선수금 / 가입자 수** — KOSIS에 없음. 공정거래위원회 보도자료 별도 수집.
- **장례식장·화장시설 수급** — 보건복지부 별도 자료. KOSIS 일부에 있으나 누락 많음.
- **고독사·무연고 사망자** — 보건복지부 고독사 실태조사 별도. KOSIS 직접 표 없음.
- **사회조사의 죽음·임종·장례 인식 문항** — KOSIS 미시 데이터 가능. 격년 수집이라
  Phase 2에서 별도 다룸.

이 네 개는 KOSIS 외부 자료이고, 별도 트랙으로 잡아야 한다. 이번 인프라에 무리해서
욱여넣지 않는다.
